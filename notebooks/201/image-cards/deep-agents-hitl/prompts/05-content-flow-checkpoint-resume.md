【主体内容】flow 左右双栏对照流程图，中间放一个手绘「checkpointer 存档桶/磁盘」图标作为枢纽（珊瑚红高亮），两栏之间画手绘弯箭头连接（暂停存档 → 恢复读取）：

左栏「第一次 invoke」（淡蓝块，自上而下）：
用户：写 /test.md → 🧠 LLM 决定调 write_file → ⛔ interrupt_on 命中，执行前拦下 ⏸ → checkpointer 存档(thread_id) → 返回 __interrupt__ 暂停。

中央 checkpointer 存档桶：标注 key = thread_id（同一门牌号）。

右栏「第二次 invoke」（薄荷绿块，自上而下）：
Command(resume=决策) → 按 thread_id 读档恢复现场 → 从 write_file 断点『续跑』 → 二向分叉：approve 放行真正写盘 / reject 拦下不执行 → 返回最终响应。
